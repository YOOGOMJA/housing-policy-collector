# Prompt Engineering 운영 플레이북

## 1) 기본 템플릿
- 역할(Role): "너는 정책/문서 일관성 검토 에이전트다"
- 목표(Goal): "누락 문서와 동기화 이슈를 찾고 수정안을 제시한다"
- 제약(Constraints): "문서 구조 유지, 과도한 범위 확장 금지"
- 출력(Output): "요약/수정 파일/검증 결과"

## 2) 권장 기법
- 명확하고 직접적인 지시 사용
- 출력 포맷(섹션/체크리스트) 고정
- 필요한 경우 소수의 예시(multishot) 제공
- 최종 출력 전 자기검증 체크리스트 적용

## 3) 실패 방지 가드레일
- 근거 없는 추정 문장 금지
- 변경 범위 밖 파일 수정 금지
- 정책/기술 문서 간 불일치 발생 시 경고 후 보류

## 4) 작업 유형별 프롬프트 스캐폴드
### A. 분석 작업
- 입력: 현재 문서 목록 + 기준 문서
- 출력: 누락 항목, 우선순위, 생성/수정 필요 문서

### B. 수정 작업
- 입력: 대상 파일 + 반영 규칙
- 출력: 파일별 변경 요약 + 교차 문서 반영 여부

### C. 리뷰 작업
- 입력: 변경 diff + 정책 문서
- 출력: 위반 가능성, 충돌 지점, 후속 조치
