# AI 가이드라인

## 1) 목적
- 이 프로젝트의 AI 에이전트 작업 품질(정확성/재현성/보안)을 높입니다.
- 문서 우선(Policy/Tech/Ops)을 기본 원칙으로 유지합니다.

## 2) 공통 원칙
- 구현 전후 관련 문서를 동기화합니다.
- 정책 위반 가능성은 즉시 경고합니다.
- 모호한 조건은 보수적으로 판단합니다.
- 코드 변경 시 린트 및 컨벤션 체크를 수행합니다.
- 문서 변경 시 YAML 파일에 한해 포맷/문법 체크를 수행합니다.
- 신규 문서 추가 시 문서 인덱스와 연관 문서 반영 누락을 확인합니다.
- 모든 문서는 한국어를 기본 작성 언어로 사용합니다(고유명사/기술용어 제외).
- PR 제목/본문/검증 결과는 한국어로 작성합니다.
- 충돌 발생 시 충돌 원인·해결 방식·재발 방지 조치를 PR 본문에 기록합니다.

## 3) 문서 변경 규칙
- 정책 문서(`docs/01-policy/*`) 변경 시, 관련 기술 문서(`docs/02-tech-wiki/*`)와 운영 문서(`docs/03-operations/*`)를 같은 작업 단위에서 검토합니다.
- AI 지침 문서(`docs/04-ai-instructions/*`)가 추가/수정되면 `docs/README.md` 인덱스를 함께 갱신합니다.
- AI 지침 문서가 변경되면 `README.md`의 "현재 원칙"과 충돌 여부를 확인합니다.

## 4) 산출물 품질 기준
- 의사결정 근거: 참조한 문서/정책 위치를 명확히 기록합니다.
- 추적 가능성: 변경사항이 어떤 요구사항/정책을 만족하는지 연결합니다.
- 운영 가능성: 실패 시 대응 절차와 관측 지표를 함께 명시합니다.
