# AGENTS.md 운영 가이드

## 1) 목적
에이전트가 저장소 규칙을 안정적으로 해석하도록 AGENTS.md 배치/관리 기준을 정의합니다.

## 2) 배치 원칙
- 루트 AGENTS.md: 저장소 공통 규칙
- 하위 AGENTS.md: 특정 디렉터리 전용 규칙
- 더 깊은 경로의 AGENTS.md가 우선 적용

## 3) 규칙 작성 원칙
- 빌드/테스트 명령을 명시합니다.
- 코드 스타일, 문서 반영 규칙, PR 규칙을 분리해 씁니다.
- 금지사항(예: 민감정보 로그 금지)은 명확한 문장으로 작성합니다.

## 4) 변경 절차
1. 규칙 변경 이유를 이슈에 기록
2. AGENTS.md 수정
3. 관련 문서(`docs/workflow.md`, `docs/README.md`) 동기화
4. 샘플 작업 1건으로 규칙 적용 검증

## 5) 점검 체크리스트
- 스코프 충돌이 없는가?
- 동일 규칙이 중복 정의되지 않았는가?
- 새 규칙이 기존 워크플로우와 모순되지 않는가?
