# Context Window 운영 플레이북

## 1) 목적
긴 문서/다중 정책 작업에서 컨텍스트 손실과 누락을 줄이기 위한 운영 규칙입니다.

## 2) 컨텍스트 예산 규칙
- 필수 컨텍스트: 현재 작업 대상 문서, 직접 의존 문서, 상위 정책 문서
- 선택 컨텍스트: 참고 예시, 과거 결정 로그, 관련 없는 운영 문서
- 원칙: "필수 먼저, 선택은 필요 시" 순서로 로딩

## 3) 문서 로딩 순서
1. 작업 지시문(시스템/개발자/사용자)
2. 변경 대상 문서
3. 상위 인덱스(`docs/README.md`) 및 연결 문서
4. 검증 대상(정책/기술/운영 교차 문서)

## 4) 긴 작업 분할 방식
- 1회차: 범위 확인 + 누락 항목 목록화
- 2회차: 문서 수정 + 연관 문서 반영
- 3회차: 최종 점검(충돌/누락/추적성)

## 5) 응답 품질 체크
- 작업 범위가 처음 요청과 일치하는가?
- 변경된 문서가 인덱스에 반영되었는가?
- 추가 문서가 있다면 교차 참조가 연결되었는가?
