# AGENTS.md

이 문서는 `housing-policy-collector` 저장소에서 코딩 에이전트가 작업할 때 따라야 할 기본 운영 규칙입니다.

## 1) 우선순위 규칙
1. 시스템 지시 > 개발자 지시 > 사용자 지시 > AGENTS.md 순서로 해석합니다.
2. 하위 디렉터리에 또 다른 `AGENTS.md`가 있으면 더 깊은 파일을 우선 적용합니다.
3. 충돌 시 더 구체적인 범위의 지시를 우선하며, 애매하면 보수적으로 처리합니다.

## 2) 문서 중심 작업 원칙
- 이 저장소는 문서 우선 프로젝트입니다. 구현/정책 변경 시 관련 문서를 함께 동기화합니다.
- 문서 추가/수정 시 최소 아래 3개를 교차 점검합니다.
  - `docs/README.md` (인덱스)
  - `docs/workflow.md` (협업/PR 체크리스트)
  - `README.md` (루트 원칙)

## 3) 컨텍스트 윈도우 운영 규칙
- 긴 작업은 "범위 확인 → 변경 → 동기화 검증" 3단계로 분할합니다.
- 필수 문맥(요청, 대상 파일, 직접 연관 문서)부터 읽고 선택 문맥은 필요 시 추가합니다.
- 답변 전 "요청 범위 밖 변경" 여부를 자기검증합니다.

## 4) 프롬프트 품질 가드레일
- 출력 포맷(요약/변경내역/검증결과)을 고정해 재현성을 높입니다.
- 근거 없는 추정이나 과도한 범위 확장을 피합니다.
- 정책/기술/운영 문서 간 불일치 발견 시 경고하고 후속 조치를 명시합니다.

## 5) 검증 규칙
- 코드 변경 시 린트/컨벤션 체크를 수행합니다.
- 문서 변경은 YAML 파일이 포함된 경우에만 문서 포매팅/문법 체크를 수행합니다.
- 문서 신규 추가 시 인덱스/참조 누락 여부를 반드시 점검합니다.

## 6) 커밋/PR 규칙
- 커밋 메시지는 Conventional Commits를 따릅니다.
- PR 본문에는 배경/작업내용/기대효과/리뷰포인트/체크리스트를 포함합니다.
