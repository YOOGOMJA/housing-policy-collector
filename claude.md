# Claude 작업 가이드

이 문서는 Claude 계열 모델을 사용할 때, 본 저장소에서 일관된 품질로 문서·정책 작업을 수행하기 위한 실행 가이드입니다.

## 1) 목표
- 긴 문맥에서도 누락 없이 변경 범위를 지키고, 문서 간 일관성을 유지합니다.
- 에이전트 작업의 재현성(같은 입력 → 유사한 결과)을 높입니다.

## 2) 권장 입력 템플릿
아래 순서로 요청을 구조화합니다.
1. 작업 목표 (무엇을 완료할지)
2. 변경 범위 (수정 가능한 파일/불가 파일)
3. 제약 조건 (포맷, 규칙, 금지사항)
4. 검증 조건 (필수 체크 명령)
5. 출력 형식 (요약, 파일별 변경, 테스트 결과)

## 3) 컨텍스트 윈도우 운용
- 우선 읽기: 지시문 + 대상 문서 + 인덱스 문서
- 이후 읽기: 직접 연관 정책/기술/운영 문서
- 보류: 관련도 낮은 배경 문서
- 긴 요청은 단계형으로 처리합니다.
  - 1단계: 누락/영향도 분석
  - 2단계: 문서 수정
  - 3단계: 교차 참조/인덱스 검증

## 4) 프롬프트 엔지니어링 체크리스트
- 지시는 명확하고 직접적으로 작성합니다.
- 원하는 출력 포맷을 예시와 함께 제공합니다.
- 분석/수정/리뷰를 분리해 요청합니다.
- 최종 응답 전에 아래를 자기점검합니다.
  - 요청 범위를 벗어난 수정이 없는가?
  - 신규 문서가 인덱스에 반영되었는가?
  - 정책/기술/운영 문서 간 충돌이 없는가?
